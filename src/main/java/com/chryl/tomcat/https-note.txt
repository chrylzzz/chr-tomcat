
HTTPS:内容
公钥和私钥:公钥和私钥组合起来是一对密钥
1.
公钥:密钥中公开的部分
私钥:密钥中不公开的部分
2.
对称加密:加密原文 与 解密原文 所使用的密钥 为同一密钥(加密和解密的密码一样)
非对称加密:加密原文 , 和 机密密文 不是同一个,但是使用的密码一对密文
3.
http:超文本传输协议
https:安全的超文本传输协议,在http基础上加上ssl层
4.
SSL:是为网络通信提供安全及数据完整性的 安全协议
5.
SSL证书 是 SSL的一个典型应用,具有服务器身份验证和数据加密的功能
6.
CA:证书颁发机构,该机构负责证书的申请的资格审查,证书颁发,证书吊销/维护...等等操作



证书的分类: 部署应用的时候 如何选择合适的证书
分类:
1.从终端上分类:
S/MIME证书:不常见,主要用于 电子邮件保护,以及服务器证明自己的身份
Server SSL证书:服务器证书,比较常用
    DV  :域名验证,签发时间快,成本低,几分钟就可以
    OV  :组织验证,签发时间长,需要验证组织的合法性(税务..)
    EV  :级别最高,时间最长,除了要验证OV的所有信息,还需要律师函等证明文件
代码签名证书:主要对代码的签名,确保代表软件的发行者



https工作原理:非对称加密
服务器有 公钥和私钥,每个用户只有 公钥,用户通过公钥发送给服务器信息,
只有服务器有私钥,才能进行解密,
